## Estrategia de Ramificación
Para este proyecto usamos el modelo **GitFlow**:
* `main`: Código estable de producción.
* `develop`: Rama para el trabajo diario e integración.
* `feature/`: Ramas para tareas específicas.

## Buenas Prácticas
1. **Mensajes de Commit:** Formato `Tipo: Descripción`

2. **Frecuencia:** Subidas constantes para trazabilidad.
3. **Colaboración:** Uso de Pull Requests entre compañeros

## Automatización
Configuramos **GitHub Actions** para validar el código automáticamente en cada subida a la rama `develop`
